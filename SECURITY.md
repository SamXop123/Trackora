# Security Policy

## Supported Versions

Only the latest release version of Trackora is supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Trackora, **please do not open a public issue.** This allows time to address the issue before public disclosure.

Instead, please report the vulnerability directly to the project maintainer:
* **Discord**: Reach out to **`dot_notsam`** on Discord.

### What to Include
When reporting, please include:
1. A clear description of the vulnerability and its potential impact.
2. Step-by-step instructions or a Proof of Concept (PoC) to reproduce the issue.
3. Information about your environment (GNOME Shell version, Fedora version, etc.).

We will acknowledge your report, verify the vulnerability, and coordinate a fix as quickly as possible.
